# Topic: Patching Virtual Machines (Azure & AWS)

## Real-World Scenario (Set the Context First)

Imagine your company has **two teams**:

| Team | Role |
|------|------|
| 🔧 **Admin Team (Us)** | Creates cloud infrastructure — VMs, networks, storage |
| 💻 **Project Team** | Developers who use the VMs to deploy their applications |

### The Problem:
- The **Project Team** holds the **login credentials** of their VMs.
- The **Admin Team** does NOT have SSH passwords or keys to those VMs.
- But the **Admin Team is still responsible** for keeping the VMs patched and secure.

### The Question:
> **"How does the Admin Team update packages on VMs they cannot directly SSH into?"**

This is exactly what we solve today — using **Azure Run Command** and **AWS Systems Manager**.

---

## 🟦 Part 1: Azure – VM Patching Without SSH

### 🔑 Key Concept: Azure VM Agent

Every Azure Linux VM has a built-in **agent** running inside it called the **Azure Linux Agent (waagent)**.

```
┌──────────────────────────────────────┐
│           Azure Portal / CLI         │
│         (Admin Team sends command)   │
└──────────────┬───────────────────────┘
               │  HTTPS / Azure API
               ▼
┌──────────────────────────────────────┐
│          Azure VM (Ubuntu)           │
│   ┌──────────────────────────────┐   │
│   │   Azure Linux Agent (waagent)│   │  ← This receives the command
│   └──────────────────────────────┘   │
│   ┌──────────────────────────────┐   │
│   │   Your Script Runs Here      │   │
│   └──────────────────────────────┘   │
└──────────────────────────────────────┘
```

> 📌 **Reference:** [Azure Linux Agent on GitHub](https://github.com/Azure/WALinuxAgent)  
> 📌 **Docs:** [Azure VM Extensions – Linux Agent](https://learn.microsoft.com/en-us/azure/virtual-machines/extensions/agent-linux)

---

### Azure VM Patching

#### Step 1: (As Project Team Member) – Create a VM

1. Go to [Azure Portal](https://portal.azure.com)
2. Create a **Free Tier Ubuntu Virtual Machine**
   - Image: `Ubuntu 22.04 LTS`
   - Size: `Standard_B1s` (Free Tier)
   - Authentication: SSH Key or Password (project team keeps this)
3. VM is now running — **project team has the credentials**

> 🔗 [Quick Create Ubuntu VM on Azure](https://learn.microsoft.com/en-us/azure/virtual-machines/linux/quick-create-portal?tabs=ubuntu)

---

#### Step 2: (As Admin Team) – Patch the VM Using Run Command

> 🎙️ *Tell students: "Now switch role. You are the Admin. You don't have SSH access. How do you patch?"*

**In Azure Portal:**

```
VM → Operations → Run Command → RunShellScript
```

**The Script to Run:**

```bash
#!/bin/bash
apt update          # Refreshes the list of available packages
apt upgrade -y      # Upgrades all installed packages automatically
```

> ✅ Azure sends this script to the **VM Agent**, which executes it inside the VM.  
> ✅ Output is shown back in the portal — no SSH needed!

---

### 📖 How Azure Run Command Works (Explain to Class)

```
Admin (Portal) ──► Azure API ──► VM Agent (waagent) ──► Shell Script Runs
                                        │
                                        └──► Output returned to Portal
```

---

### 💡 What is `apt update` vs `apt upgrade`?

| Command | What It Does | Example Output |
|---------|-------------|----------------|
| `apt update` | Downloads the **latest package list** from internet repos. Does NOT install anything. | `Get:1 http://archive.ubuntu.com... 45.2 MB fetched` |
| `apt upgrade -y` | Installs all **available updates** for installed packages. `-y` means "yes to all". | `Setting up openssh-server (1:9.6p1) ...` |
| `apt upgrade` (without -y) | Same but **asks for confirmation** before installing. | `Do you want to continue? [Y/n]` |

> 🎙️ *Classroom tip: Run both commands live in terminal. Show students the difference in output.*

## 🟧 Part 2: AWS – VM Patching Without a Key Pair

### 🔑 Key Concept: Why AWS is Different from Azure

| Feature | Azure | AWS |
|---------|-------|-----|
| Built-in agent in VM | ✅ Yes (waagent) | ❌ No (must be installed) |
| Default username | You set it during VM creation | Loaded from the AMI image (e.g., `ec2-user`, `ubuntu`) |
| Authentication | Password or SSH Key | **SSH Key only** (no password by default) |
| Remote script execution | Azure Run Command (via agent) | AWS Systems Manager (SSM Agent) |

> 🎙️ *Tell students: "In AWS, VMs (EC2 instances) are more locked down. There's no built-in agent. AWS provides a separate service called Systems Manager to compensate."*

---

### 🔗 Two Ways to Connect to EC2 Without SSH Keys

#### Method 1: EC2 Instance Connect

```
AWS Console → EC2 → Select Instance → Connect → EC2 Instance Connect
```

- Opens a **browser-based terminal** directly in the AWS Console
- No SSH key needed
- Works for **Amazon Linux 2, Ubuntu** AMIs
- The admin needs **IAM permission**: `ec2-instance-connect:SendSSHPublicKey`

**How it works:**

```
Admin (Browser) ──► AWS API ──► Temporary SSH Key pushed to EC2 ──► Shell Access
                                        (valid for 60 seconds only)
```

> 📌 **Best for:** Quick access, troubleshooting, one-time commands

---

#### Method 2: Session Manager (AWS Systems Manager)

```
AWS Console → Systems Manager → Session Manager → Start Session → Select EC2
```

- Fully **agentless from admin side** — no SSH, no keys, no open port 22!
- Requires **SSM Agent** to be installed inside the EC2 instance
- All session activity is **logged to CloudWatch / S3**
- Admin needs IAM role with `ssm:StartSession` permission

**How it works:**

```
Admin (Console/CLI)
        │
        ▼
AWS Systems Manager ──► SSM Agent (inside EC2) ──► Shell Access
        │
        ▼
  Session logged to CloudWatch Logs
```

> 📌 **Best for:** Production environments, audit-required access, no need to open port 22

---

### 🧪 Classroom Demo – AWS EC2 Patching via Session Manager

#### Step 1: Verify SSM Agent is Running on EC2

```bash
# Check SSM Agent status (inside EC2 via Session Manager)
sudo systemctl status amazon-ssm-agent
```

Expected output:
```
● amazon-ssm-agent.service - amazon-ssm-agent
   Active: active (running) since ...
```

#### Step 2: Run Package Update

**For Ubuntu EC2:**
```bash
sudo apt update && sudo apt upgrade -y
```

**For Amazon Linux 2 / RedHat / CentOS:**
```bash
sudo dnf update -y
```

---

### 📦 Package Managers Comparison

> 🎙️ *This is important — students need to know which command to use on which OS.*

| OS / Distribution | Package Manager | Update Command |
|------------------|----------------|----------------|
| Ubuntu / Debian | `apt` | `apt update && apt upgrade -y` |
| Amazon Linux 2023 | `dnf` | `dnf update -y` |
| RedHat / CentOS 8+ | `dnf` | `dnf update -y` |
| RedHat / CentOS 7 | `yum` | `yum update -y` |
| SUSE Linux | `zypper` | `zypper update -y` |

---

## 🔴 Difference: `dnf update` vs `apt update`

| Feature | `apt` (Ubuntu/Debian) | `dnf` (RedHat/Amazon Linux) |
|---------|----------------------|----------------------------|
| Full update command | `apt update && apt upgrade -y` | `dnf update -y` |
| Refresh package list | `apt update` (separate step) | `dnf update` (does both together) |
| Package format | `.deb` | `.rpm` |
| Dependency resolution | Yes | Yes (better than older `yum`) |
| Successor of | — | `yum` |

## 🧠 Concept Check – Quiz Questions

> 🎙️ *Ask these questions to the class before wrapping up.*

**Q1:** Why can't the Admin team just SSH into the VM to patch it?
> *Answer: Because the project team holds the credentials/SSH keys. Admin team doesn't have them.*

**Q2:** What is the Azure VM Agent and why is it important?
> *Answer: It's a process (waagent) running inside every Azure VM that listens for commands from Azure API and executes them — enabling remote script execution without SSH.*

**Q3:** What is the difference between EC2 Instance Connect and Session Manager?
> *Answer: EC2 Instance Connect is browser-based SSH (temporary key, port 22 needed). Session Manager uses SSM Agent — no port 22, fully auditable.*

**Q4:** What command refreshes the package list on Ubuntu without installing anything?
> *Answer: `apt update`*

**Q5:** Which package manager does RedHat 9 use?
> *Answer: `dnf`*
