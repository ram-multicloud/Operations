# Monitoring
Monitoring is the continuous process of **collecting, tracking, and analyzing metrics** from infrastructure and applications to ensure they are healthy, available, and performing as expected.

A **metric** is dimensional data consisting of:
- **Name** — what is being measured (e.g., CPUUtilization)
- **Value** — the measured quantity at a point in time
- **Timestamp** — when the measurement was recorded

Monitoring helps teams:
- Detect issues before they impact users
- Set up **alerts/alarms** for threshold breaches
- Take **corrective actions** automatically or via notifications



## Monitoring Infrastructure with AWS

### Service: Amazon CloudWatch

- AWS provides a centralized monitoring service called **CloudWatch**
- **Basic monitoring** is provided for free for AWS resources
- CloudWatch collects metrics from services like EC2, S3, RDS, and more

### Key Concepts

| Concept | Description |
|---|---|
| **Metric** | Dimensional data with name, value, and timestamp |
| **Alarm** | A condition defined on a metric to trigger notifications or actions |
| **Namespace** | A container for CloudWatch metrics (e.g., AWS/EC2) |

### CloudWatch Alarm States

| State | Meaning |
|---|---|
| `OK` | Metric is within the defined threshold |
| `ALARM` | Metric has breached the threshold |
| `INSUFFICIENT_DATA` | Not enough data to determine the state |

### Practice Steps

1. Create an **EC2 instance** → view its monitoring tab
2. Create an **S3 bucket** → observe storage metrics
3. Create an **RDS instance** → observe DB metrics
4. View the **centralized CloudWatch dashboard** for all resources
5. Create a **CloudWatch Alarm** with a condition and link it to a **notification service (SNS)**


## Monitoring Infrastructure with Azure

### Service: Azure Monitor

- Azure provides **Azure Monitor** as its centralized monitoring service
- It collects metrics and logs from all Azure resources

### Practice Steps

1. Create a **Resource Group** containing:
   - Virtual Machine (VM)
   - Storage Account
   - Azure SQL Database
2. Open **Azure Monitor** to view metrics across all resources in the group
