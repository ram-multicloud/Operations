## Azure Update Manager · Azure VM Troubleshooting · AWS EC2 Troubleshooting

## 1. Update Manager in Azure (VM Patching)

Azure **Update Manager** is used to patch virtual machines at scale.

### Steps to Get Started

1. Create two VMs (as discussed in class)
2. Navigate to **Update Manager** in the Azure Portal
3. Configure schedules and apply updates

### Important Note

> ⚠️ For **production systems**, automatic reboot is **not recommended**.  
> Always reboot manually during a planned maintenance window.


## 2. Azure VM Troubleshooting

### 2.1 When the Agent is Connected

If the VM agent is responsive, you can troubleshoot using tools like:

- **Run Command** — execute scripts directly on the VM from the portal without SSH/RDP

---

### 2.2 Boot Diagnostics

Boot Diagnostics provides two key views to help diagnose VM startup issues:

| Feature | Description |
|---|---|
| **Screenshot** | Captures the current screen of the VM (useful for seeing boot errors visually) |
| **Serial Log** | Text-based log of the VM boot process and kernel messages |

#### How to Access
- Go to your VM in the Azure Portal
- Navigate to **Help → Boot Diagnostics**
- Choose **Screenshot** or **Serial Log**

---

### 2.3 Serial Console

The **Serial Console** is a low-level management interface that lets you connect directly to a VM's boot and system console — like plugging a monitor and keyboard into a physical server in a data center.

#### Key Characteristics

- Operates **below the network layer**
- Works even when **SSH / RDP is failing**
- Can fix:
  - Boot or kernel errors
  - Network misconfigurations
  - Lost passwords (with proper setup)

#### How to Access

1. Ensure **Boot Diagnostics** is enabled on the VM
2. Go to VM → **Help → Serial Console**
3. Make sure the VM has a **user with password-based authentication** configured

---

## 3. AWS EC2 Troubleshooting

AWS EC2 provides equivalent troubleshooting features to Azure, accessible from the EC2 Console:

| Feature | AWS Equivalent | Azure Equivalent |
|---|---|---|
| VM Screenshot | Get Instance Screenshot | Boot Diagnostics → Screenshot |
| System Log | Get System Log | Boot Diagnostics → Serial Log |
| Low-level console | EC2 Serial Console | Serial Console |

### How to Access in AWS

1. Go to **EC2 Console → Instances**
2. Select your instance
3. Click **Actions → Monitor and Troubleshoot**
4. Choose from:
   - **Get Instance Screenshot**
   - **Get System Log**
   - **EC2 Serial Console**


## Summary

| Topic | Tool | When to Use |
|---|---|---|
| Patch VMs at scale | Azure Update Manager | Regular OS patching on schedule |
| Run scripts remotely | Run Command | When agent is connected |
| View VM boot screen | Boot Diagnostics (Screenshot) | VM not responding, visual error needed |
| Read boot logs | Boot Diagnostics (Serial Log) | Kernel / OS boot failure |
| Connect below network | Serial Console | SSH/RDP down, kernel or network misconfiguration |
| AWS equivalent | EC2 Monitor & Troubleshoot | Same scenarios as above on AWS |
